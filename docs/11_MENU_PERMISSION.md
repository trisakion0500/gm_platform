---

# 1. 역할 정의

| Role Code | Role Name   | 설명                                        |
| --------- | ----------- | ------------------------------------------- |
| 10        | SUPER_ADMIN | 모든 회사 / 프로젝트 / API 접근 가능        |
| 20        | DEVELOPER   | API 정의 관리, 승인, 실행                   |
| 30        | APPROVER    | API 승인 / 반려                             |
| 40        | OPERATOR    | API 실행 요청 및 결과 조회                  |

---

# 2. 관리 메뉴

시스템 및 조직 운영에 필요한 기능. 주로 SUPER_ADMIN이 사용한다.

## 2.1 회사 관리

| 메뉴           | SUPER_ADMIN     | DEVELOPER        | APPROVER        | OPERATOR        |
| -------------- | :-----------:   | :--------------: | :-------------: | :-------------: |
| 회사 목록 조회 | O (전체)        | O (본인 회사만)  | O (본인 회사만) | O (본인 회사만) |
| 회사 상세 조회 | O (전체)        | O (본인 회사만)  | O (본인 회사만) | O (본인 회사만) |
| 회사 등록      | O               | -                | -               | -               |
| 회사 수정      | O               | -                | -               | -               |

> APPROVER / OPERATOR 의 목록·상세 조회는 "회사 관리" 관리 화면 접근이 아니라, 헤더의 회사 표시·콤보박스 구성을 위한 API 레벨 허용이다. 이 두 역할은 회사 관리 화면(SCR-010~012) 자체에는 접근하지 않는다.

---

## 2.2 프로젝트 관리

| 메뉴               | SUPER_ADMIN | DEVELOPER      | APPROVER       | OPERATOR       |
| ------------------ | :---------: | :------------: | :------------: | :------------: |
| 프로젝트 목록 조회 | O (전체)    | O (역할보유만) | O (역할보유만) | O (역할보유만) |
| 프로젝트 상세 조회 | O (전체)    | O (역할보유만) | O (역할보유만) | O (역할보유만) |
| 프로젝트 등록      | O           | -              | -              | -              |
| 프로젝트 수정      | O           | -              | -              | -              |

> SUPER_ADMIN 외에는 "본인 소속 회사의 프로젝트 전체"가 아니라 "본인이 활성 `user_role`을 가진 프로젝트만" 반환된다(`user_role.status=1` 기준). 같은 회사 소속이어도 role 미배정 프로젝트는 조회되지 않는다.
> APPROVER / OPERATOR 의 목록·상세 조회는 관리 목적이 아니라 API 선택, 실행 이력 조회, 승인 대기 목록 진입 시 프로젝트를 선택하기 위한 탐색 경로로 오픈된 것이다.
